# 🖼️ VK Caption AI

![Tauri 2](https://img.shields.io/badge/Tauri-2.0-blue?logo=tauri)
![Vue 3](https://img.shields.io/badge/Vue-3.x-green?logo=vue.js)
![Naive UI](https://img.shields.io/badge/Naive%20UI-2.x-green)
![LMStudio](https://img.shields.io/badge/LMStudio-Vision-orange)

Десктопное приложение на базе Tauri + Vue 3, которое использует локальные Vision-модели через LM Studio для двух независимых задач:

- **Генерация описаний** к фотографиям в альбомах VK
- **Сортировка локальных фото** по папкам на основе содержимого

Всё работает локально — никаких облачных API, никакой утечки данных.

---

## 🚀 Быстрый старт

### Требования

- **Node.js** >= 24.14
- **Rust** >= 1.70
- **LM Studio** с загруженной Vision-моделью (`zai-org/glm-4.6v-flash`, `google/gemma-3-12b`, `google/gemma-4-26b-a4b`)

### Установка и запуск

```bash
git clone https://github.com/your-username/vk-caption-ai.git
cd vk-caption-ai
npm install
npm run tauri:dev
```

### Сборка

```bash
npm run tauri:build
```

Готовый бинарник будет в `src-tauri/target/release/bundle`.

## 📸 Генерация описаний для VK

Вкладка для автоматической расстановки описаний (alt-текстов) к фотографиям в альбомах VK.

### Требования

- **VK Access Token** с правами `photos` — получите на [vkhost.github.io](https://vkhost.github.io/)

### Настройка

В разделе **«Настройки»** заполните:

- **VK Token** — персональный токен доступа
- **VK Album ID** — числовой ID альбома
- **LMStudio Model** — идентификатор модели
- **Processing Mode** — выберите «Ручной», если хотите проверять каждое описание перед публикацией

### Режимы работы

**Автоматический** — модель обходит весь альбом и сохраняет описания без остановки.

**Ручной (интерактивный)** — после каждого фото открывается модальное окно, где можно:
- отредактировать сгенерированный текст
- перегенерировать описание с комментарием для модели
- пропустить фото
- перейти напрямую к фото в VK

### Дополнительные возможности

- Выбор направления обхода альбома (сначала новые или старые)
- Опция повторной обработки фото, у которых уже есть описание
- Настраиваемый системный промпт
- Undo — возвращает предыдущее описание в VK
- Автоматические повторы при сбоях API

### Про стикеры

По умолчанию системный промпт настроен на генерацию кратких тегов для стикеров: модель возвращает JSON-массив, где первый элемент — текст с картинки, остальные — поисковые ключевые слова (эмоции, ситуации).

## 🗂️ Сортировка фото по папкам

Вкладка для автоматической раскладки локальных изображений по подпапкам — модель смотрит на каждый файл и сама решает, в какую папку его переместить.

### Как использовать

1. Укажите **папку-источник** — откуда брать файлы
2. Опционально задайте **папку для подкаталогов** (по умолчанию совпадает с источником)
3. Добавьте **целевые папки** через теги — например: `Люди`, `Природа`, `Скриншоты`
4. Напишите **правила классификации** в свободной форме — модель будет им следовать
5. Нажмите **«Запустить»**

Если файл не подходит ни под одну категорию, модель возвращает `SKIP`. Такие файлы можно оставить на месте или автоматически переместить в отдельную папку (например, `_unsorted`).

### Дополнительные параметры

- **Переопределение модели** — использовать другую модель LMStudio только для этой задачи
- **Своя температура** — настройка в диапазоне 0–2
- **Папка для нераспознанных** — имя подпапки для файлов со статусом SKIP

### Статусы файлов

| Статус | Описание |
|---|---|
| Ожидание | Файл ещё не обработан |
| Обработка | Модель анализирует изображение |
| Перемещено | Файл перемещён в целевую папку |
| Пропущено | Модель вернула SKIP |
| Ошибка | Сбой при классификации или перемещении |
